# /office-hours — Product Strategy

Use this workflow to brainstorm new features, clarify user pain points, and challenge assumptions BEFORE any code is written.

## Phase 1: Context Gathering
1. Review the current project state (read `CLAUDE.md`, `ARCHITECTURE.md`).
2. Ask the user for the core objective of the new feature/change.

## Phase 2: Forcing Questions
Ask the user (or answer yourself based on context) these 5 "GStack Forcing Questions":
- "What is the single most important user outcome for this change?"
- "What is the simplest version of this that we can ship in 2 hours?"
- "How does this impact the existing patient/admin workflow?"
- "What is the biggest technical or product risk we are taking here?"
- "How will we know if this feature is a success (what metric)?"

## Phase 3: Conclusion
1. Summarize the refined product vision.
2. Direct the user to run `/plan-ceo-review` to formalize the scope.
